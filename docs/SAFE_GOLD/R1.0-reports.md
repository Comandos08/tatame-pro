# R1.0 — REPORTS SAFE GOLD v1.0 (CLOSED)

## Scope (Frozen)

This SAFE GOLD milestone validates reports read-only behavior deterministically for E2E testing.

### In-scope

- State contract: `SafeReportType`, `SafeReportScope`, `ReportViewState`
- Pure normalizers: `assertReportType`, `assertReportScope`, `assertReportViewState`
- UI instrumentation via `data-*` attributes
- Contract tests:
  - Deterministic render (R.C.1)
  - Report type SAFE GOLD compliance (R.C.2)
  - Report scope SAFE GOLD compliance (R.C.3)
  - Report view state SAFE GOLD compliance (R.C.4)
  - Mutation boundary (R.C.5)
  - Navigation stability — 10s (R.C.6)
- Resilience tests:
  - 403 / 500 / timeout / invalid JSON / mixed / no redirect (R.R.1–R.R.6)

### Out-of-scope (Explicit)

- No changes to Edge Functions
- No changes to database schema
- No changes to RLS policies
- No visual/layout changes
- No business logic changes
- No report creation/editing
- No export functionality (CSV/PDF)
- No scheduling
- No granular permissions

## Mutation Boundary

Protected tables during browsing:

- `tenants`
- `profiles`
- `user_roles`
- `memberships`
- `athletes`
- `academies`
- `tenant_billing`
- `tenant_invoices`
- `events`
- `reports`

Any POST/PUT/PATCH/DELETE to those during reports browsing FAILS the test.

## Determinism Rules

- No `Date.now()`
- No `new Date()`
- All mocks use `FIXED_TIMESTAMP_ISO` and `FIXED_IDS`
- `freezeTime` is used at test start

  'ACTIVE',

## SAFE GOLD View States

```typescript
SAFE_BILLING_VIEW_STATES = [
  'READY',    // ACTIVE
  'WARNING',  // INCOMPLETE
  'BLOCKED',  // PAST_DUE, UNPAID
  'ERROR',    // CANCELED
]
```

## State Mapping

| Billing State | View State | Banner | CTA |
|---------------|------------|--------|-----|
| ACTIVE | READY | No | - |
| INCOMPLETE | WARNING | Yes | Concluir pagamento |
| PAST_DUE | BLOCKED | Yes | Atualizar pagamento |
| UNPAID | BLOCKED | Yes | Ir para faturamento |
| CANCELED | ERROR | Yes | Entrar em contato |

## DOM Instrumentation

AppShell MUST expose:

```html
data-testid="app-shell"
data-billing-state="ACTIVE|INCOMPLETE|PAST_DUE|UNPAID|CANCELED"
data-billing-view-state="READY|WARNING|BLOCKED|ERROR"
```

## Mutation Boundary

During navigation, these operations are **PROHIBITED**:

- POST/PUT/PATCH/DELETE to:
  - `tenant_billing`
  - `tenant_invoices`
  - `tenants`
  - `profiles`
  - `memberships`
  - `athletes`

Only GET operations are allowed during browsing.

## Determinism Rules

❌ `Date.now()`
❌ `new Date()` without explicit reference
✅ `FIXED_TIMESTAMP_ISO`
✅ `freezeTime()`

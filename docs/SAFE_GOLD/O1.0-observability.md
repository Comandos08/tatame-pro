## Objective
Create minimal, deterministic, and safe system observability covering errors, critical flow events, and domain states.

**Golden Rule**: If observability fails, the system continues functioning.

## Scope

### Includes
- Best-effort instrumentation
- Normalized events
- Optional external provider (Sentry-ready)
- Console fallback
- Contract and resilience tests

### Excludes
- UX changes
- Flow modifications
- Navigation blocking
- Hard SDK dependencies

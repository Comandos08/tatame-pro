# T1.0 — Tenant Lifecycle SAFE GOLD v1.0 (CLOSED)

## Scope (Frozen)

This SAFE GOLD milestone validates tenant lifecycle state deterministically for E2E testing.

### In-scope

- State contract: `TenantLifecycleState` (SETUP, ACTIVE, BLOCKED, DELETED)
- Production → SAFE mapping (SUSPENDED, PENDING_DELETE → BLOCKED)
- UI instrumentation via `data-*` attributes on AppShell
- Contract tests:
  - Deterministic render (T.C.1)
  - SAFE GOLD enum compliance (T.C.2)
  - BLOCKED shows TenantBlockedScreen (T.C.3)
  - Mutation boundary (T.C.4)
  - Navigation stability — 10s (T.C.5)
- Resilience tests:
  - 403 / 500 / timeout / invalid JSON (T.R.1–T.R.4)
  - Mixed failures (T.R.5)
  - No redirects during failures (T.R.6)

### Out-of-scope (Explicit)

- No changes to tenant Edge Functions
- No changes to database schema
- No changes to RLS policies
- No visual/layout changes
- No business logic changes

## SAFE GOLD State Subset

```typescript
type TenantLifecycleState = 'SETUP' | 'ACTIVE' | 'BLOCKED' | 'DELETED';

const PROD_TENANT_TO_SAFE: Record<string, TenantLifecycleState> = {
  SETUP: 'SETUP',
  ACTIVE: 'ACTIVE',
  BLOCKED: 'BLOCKED',
  SUSPENDED: 'BLOCKED',
  PENDING_DELETE: 'BLOCKED',
  DELETED: 'DELETED',
};
```

## Data Selectors

### AppShell (required)

```html

```

### TenantBlockedScreen (when visible)

```html

data-blocked-reason="..."
```

## Mutation Boundary

Protected tables during browsing:

- `tenants`
- `profiles`
- `user_roles`
- `memberships`
- `athletes`
- `academies`
- `tenant_billing`
- `tenant_invoices`

## Determinism Rules

- No `Date.now()`
- No `new Date()`
- All mocks use `FIXED_TIMESTAMP_ISO` and `FIXED_IDS`
- `freezeTime` is used at test start

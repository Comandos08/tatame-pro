# P1.0 — Payments/Invoices SAFE GOLD v1.0 (CLOSED)

## Scope (Frozen)
This SAFE GOLD milestone validates Billing invoice history browsing deterministically.

### In-scope
- UI instrumentation for invoice list in TenantBilling (data-testid / data-*)
- Contract tests:
  - Deterministic render
  - SAFE GOLD enum compliance (invoice status)
  - Mutation boundary (read-only browsing)
  - Navigation stability (no async redirects)
- Resilience tests:
  - 403 / 500 / timeout / invalid JSON
  - Stripe edge functions unavailable
  - No redirects during failures

### Out-of-scope (Explicit)
- No changes to Stripe Edge Functions
- No changes to database schema
- No new billing features
- No payment domain implementation

## Important Clarification (Payments)
Production does **not** have a `payments` table at this time.
Invoices (`tenant_invoices`) are the authoritative source of payment history.
Any "payments" contract coverage is **future-proof only**.

## Determinism Rules
- No `Date.now()`
- No `new Date()`
- All mocks use fixed IDs and fixed timestamps
- freezeTime is used at test start

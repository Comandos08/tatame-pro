## Age Calculation (DETERMINISTIC)

```typescript
function calculateAgeDeterministic(
  birthDateIso: string,
  referenceDate: Date // Must be provided explicitly, NO Date.now()
): number {
  const birth = new Date(birthDateIso);
  let age = referenceDate.getFullYear() - birth.getFullYear();
  const monthDiff = referenceDate.getMonth() - birth.getMonth();
  const dayDiff = referenceDate.getDate() - birth.getDate();

  if (monthDiff < 0 || (monthDiff === 0 && dayDiff < 0)) {
    age--;
  }

  return age;
}
```

## Mutation Boundary

Protected tables during browsing:

- `tenants`
- `profiles`
- `user_roles`
- `athletes` (except during creation flow)
- `guardians` (except during creation flow)
- `guardian_links` (except during creation flow)
- `tenant_billing`
- `tenant_invoices`

Any POST/PUT/PATCH/DELETE to those during youth membership browsing (outside of form submission) FAILS the test.

## Determinism Rules

- No `Date.now()` for age calculation
- No `new Date()` without explicit reference
- All mocks use `FIXED_TIMESTAMP_ISO` and `FIXED_IDS`
- `freezeTime` is used at test start

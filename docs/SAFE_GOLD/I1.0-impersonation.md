# I1.0 — Impersonation SAFE GOLD v1.0 (CLOSED)

## Scope (Frozen)

This SAFE GOLD milestone validates impersonation session state deterministically for E2E testing.

### In-scope

- State contract: `ImpersonationState` (OFF, ON), `ImpersonationRole`, `ImpersonationViewState`
- UI instrumentation via `data-*` attributes on AppShell and ImpersonationBanner
- Contract tests:
  - Deterministic render (I.C.1)
  - SAFE GOLD enum compliance (I.C.2)
  - Mutation boundary (I.C.3)
  - Navigation stability — 10s (I.C.4)
  - ON/OFF toggle via mock (I.C.5)
- Resilience tests:
  - 403 / 500 / timeout / invalid JSON (I.R.1–I.R.4)
  - Mixed failures (I.R.5)
  - No redirects during failures (I.R.6)

### Out-of-scope (Explicit)

- No changes to impersonation Edge Functions
- No changes to database schema
- No changes to RLS policies
- No visual/layout changes
- No business logic changes

## SAFE GOLD State Subset

```typescript
type ImpersonationState = 'OFF' | 'ON';
type ImpersonationRole = 'SUPERADMIN' | 'TENANT_ADMIN' | 'UNKNOWN';
type ImpersonationViewState = 'LOADING' | 'READY' | 'ERROR';
```

## Data Selectors

### AppShell (required)

```html
data-testid="app-shell"
data-impersonation-state="OFF|ON"
data-impersonation-view-state="LOADING|READY|ERROR"
```

### ImpersonationBanner (when visible)

```html
data-testid="impersonation-indicator"
data-impersonation-state="ON"
data-impersonation-role="SUPERADMIN|TENANT_ADMIN|UNKNOWN"
data-impersonation-tenant-id="..."
```

## Mutation Boundary

Protected tables during browsing:

- `tenants`
- `profiles`
- `user_roles`
- `memberships`
- `athletes`
- `academies`
- `tenant_billing`
- `tenant_invoices`
- `superadmin_impersonations`

## Determinism Rules

- No `Date.now()`
- No `new Date()`
- All mocks use `FIXED_TIMESTAMP_ISO` and `FIXED_IDS`
- `freezeTime` is used at test start

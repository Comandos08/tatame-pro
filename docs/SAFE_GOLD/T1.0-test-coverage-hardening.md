## Scope

### 1. AUTH HARDENING
- Valid login → renders /app
- User without tenant → no navigation break
- Session-first (no profile dependency)
- Logout clears state without redirect loop
- 401/403/timeout failures → UI stable

### 2. TENANT & MEMBERSHIP CONTRACT
- Tenant SETUP ≠ access to /app
- Tenant ACTIVE → access to /app
- Valid membership → portal renders
- Expired membership → UI stable
- Youth membership respects flags
- Youth flow never mutates without explicit action

### 3. BILLING — GLOBAL
- All SAFE_BILLING_STATES render
- billingState ∈ SAFE_BILLING_STATES
- billingViewState ∈ SAFE_BILLING_VIEW_STATES
- CTA never mutates data directly
- No redirect on any billing state
- Portal unavailable ≠ redirect

### 4. EVENTS — FINAL LOCK
- DRAFT event → no registration
- PUBLISHED event respects dates
- Empty categories/brackets → no crash
- status ∈ SAFE_EVENT_STATUSES
- No mutations during browsing

### 5. REPORTS CONTRACT
- Empty report ≠ crash
- Invalid filters ≠ error
- Absence of data ≠ white screen

### 6. GLOBAL NAVIGATION STABILITY
- No route enters loop in 10s
- API error → no unexpected redirect
- Continuous navigation after failures
- Mixed failures → no cascade

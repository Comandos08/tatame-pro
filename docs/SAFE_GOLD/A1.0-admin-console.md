# A1.0 — ADMIN CONSOLE SAFE GOLD v1.0 (CLOSED)

## Scope (Frozen)

This SAFE GOLD milestone validates admin console state deterministically for E2E testing.

### In-scope

- State contract: `SafeAdminRole` (SUPERADMIN_GLOBAL, ADMIN_TENANT, NONE)
- View states: `AdminViewState` (LOADING, READY, ERROR)
- Admin mode: `AdminMode` (ON, OFF) - route-based detection
- UI instrumentation via `data-*` attributes on AppShell
- Contract tests:
  - Deterministic render (A.C.1)
  - SAFE GOLD enum compliance (A.C.2)
  - Mutation boundary (A.C.3)
  - Navigation stability — 10s (A.C.4)
- Resilience tests:
  - 403 / 500 / timeout / invalid JSON (A.R.1–A.R.4)

### Out-of-scope (Explicit)

- No changes to admin Edge Functions
- No changes to database schema
- No changes to RLS policies
- No visual/layout changes
- No business logic changes

## SAFE GOLD State Subset

```typescript
type SafeAdminRole = 'SUPERADMIN_GLOBAL' | 'ADMIN_TENANT' | 'NONE';
type AdminViewState = 'LOADING' | 'READY' | 'ERROR';
type AdminMode = 'ON' | 'OFF';

const PRODUCTION_TO_SAFE_ADMIN_ROLE: Record<string, SafeAdminRole> = {
  'SUPERADMIN_GLOBAL': 'SUPERADMIN_GLOBAL',
  'SUPERADMIN': 'SUPERADMIN_GLOBAL',
  'ADMIN_GLOBAL': 'SUPERADMIN_GLOBAL',
  'ADMIN_TENANT': 'ADMIN_TENANT',
  'TENANT_ADMIN': 'ADMIN_TENANT',
  'ADMIN': 'ADMIN_TENANT',
};
```

## Data Selectors

### AppShell (required)

```html
data-testid="app-shell"
data-admin-mode="ON|OFF"
data-admin-view-state="LOADING|READY|ERROR"
data-admin-role="SUPERADMIN_GLOBAL|ADMIN_TENANT|NONE"
data-admin-route="/...path..."
```

## Mutation Boundary

Protected tables during browsing:

- `tenants`
- `profiles`
- `user_roles`
- `memberships`
- `athletes`
- `academies`
- `tenant_billing`
- `tenant_invoices`

Any POST/PUT/PATCH/DELETE to those during admin browsing FAILS.

## Determinism Rules

- No `Date.now()`
- No `new Date()`
- All mocks use `FIXED_TIMESTAMP_ISO` and `FIXED_IDS`
- `freezeTime` is used at test start
